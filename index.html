Key Issues Found

No Meta Descriptions on Any Page

Google has no guidance on how to describe your site in search results. It will guess—and usually guess poorly. Every page needs a unique, keyword-rich meta description.

Multiple H1 Tags & Weak Heading Structure

The homepage has 5 H1 tags (Google expects exactly 1). Interior pages use generic headings like "Menu" and "Contact" instead of keyword-rich headings that help you rank.

No Sitemap, No Robots.txt

Google has no roadmap to your site's pages. A sitemap tells search engines exactly what to crawl and index.

Missing Open Graph Tags

When someone shares your site on Facebook or in a text message, it shows a generic preview instead of your restaurant's name, description, and a beautiful food photo.

Generic Image Alt Text

Gallery images are labeled "Photo 1," "Photo 2," etc. These should describe the food and include keywords like "wood fired pizza Branford CT" to help with Google Image search.

Incomplete Restaurant Schema

There's some structured data, but it's missing cuisine type, hours, menu link, price range, and other fields that help Google show rich results for your restaurant.

Section 2

The Bigger Picture

The SEO issues above aren't just checklist items—they're symptoms of a deeper problem with how the site is built.

Your Site Runs on a JavaScript Framework

Doma On Main is built with Next.js and Chakra UI—a React-based framework designed for web applications, not restaurant websites. This creates several ongoing problems:

Problem What It Means for You
Heavier page weight The browser downloads a full JavaScript application just to show your menu. More code = slower load times, especially on phones.
Framework dependency Updates to Next.js or Chakra UI can break things. You're dependent on a developer who knows React to make even simple text changes.
Harder to optimize for SEO Adding meta tags, schema markup, and proper heading structure requires editing React components—more complex and expensive than plain HTML.
Hosting complexity Next.js typically requires a Node.js server or a specialized host like Vercel. A static site can be hosted virtually anywhere, often for free.
Over-engineered for the job A restaurant site needs to load fast, look great, and rank well. It doesn't need a JavaScript application framework.

The bottom line: Your current site was built with tools meant for software, not restaurants. The right tool for this job is clean, hand-coded HTML & CSS—faster to load, easier to maintain, and dramatically better for SEO.
